# 動作確認手順書: Step 5 + Repositoryリファクタリング

## 変更範囲
- `works.$workId` のaction分岐をUse Caseに抽出
- Repositoryをファクトリ関数からモジュールレベルシングルトンに変更
- バリデーションロジックをUse Case層に移動
- `app/adapters/resultToEither.ts` 削除

## 前提
```bash
npm run db:setup  # DBが初期化済みであること
npm run dev        # 開発サーバー起動中
```

---

## 1. 作品情報編集（edit）

### 手順
1. 任意の作品詳細ページにアクセス（`/works/{id}`）
2. 「編集」ボタンをクリック
3. タイトルを変更して送信
4. 成功メッセージが表示され、タイトルが更新されていることを確認

### エラーケース
- タイトルを空にして送信 → バリデーションエラー
- 数値フィールドに文字列を入力 → バリデーションエラー

---

## 2. 視聴設定編集（watch-settings-edit）

### 手順
1. ログイン状態で作品詳細ページにアクセス
2. 視聴設定セクションで以下を入力：
   - `days`: 0
   - `hour_min`: 00:30
   - `url`: 任意のURL
3. 送信して成功メッセージを確認
4. 設定が保存されていることを確認（ページ再読込後も保持）

### エラーケース
- `days` に文字列 → バリデーションエラー
- `hour_min` に不正な形式（例: `abc`）→ バリデーションエラー
- `hour_min` を空 → バリデーションエラー

---

## 3. 視聴登録（subscribe）

### 手順
1. ログイン状態で作品詳細ページにアクセス
2. 「視聴登録」ボタンをクリック
3. 成功メッセージが表示されることを確認
4. トップページ（`/`）にアクセスし、登録作品として表示されることを確認

### エラーケース
- 未ログイン状態で実行 → ログインリダイレクト
- 既に登録済みの作品 → エラー（409または400）

---

## 4. 視聴登録解除（unsubscribe）

### 手順
1. ログイン状態で、登録済みの作品詳細ページにアクセス
2. 「登録解除」ボタンをクリック
3. 成功メッセージが表示されることを確認
4. トップページから該当作が消えていることを確認

---

## 5. エピソード削除（delete）

### 手順
1. 作品詳細ページでエピソード一覧を表示
2. 任意のエピソードの削除ボタンをクリック（2話以降を推奨）
3. 成功メッセージが表示されることを確認
4. エピソードが削除され、話数が詰め直されていることを確認

---

## 6. エピソード追加（addEpisodes）

### 手順
1. 作品詳細ページでエピソード追加フォームを表示
2. 日付を入力（カンマ区切りで複数可）
3. オフセット番号を入力
4. 送信して成功メッセージを確認
5. エピソードが追加されていることを確認

---

## 7. 作品登録（create.tsx - シングルトン変更の影響確認）

### 手順
1. `/create` にアクセス
2. 単一作品登録フォームで入力：
   - タイトル: テスト作品
   - 公開日: 任意の日付
3. 送信してリダイレクトされることを確認
4. 作品詳細ページで内容を確認

### バルク登録
1. `/create` で「複数入力する」をクリック
2. 複数の作品を入力
3. 送信して成功することを確認
